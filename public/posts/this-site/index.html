This meta-article briefly discusses this static site focused on long-form, long duration content. The features and design are very heavily inspired by gwern.net, which has been used as a template in form and function. The intention is to provide a similar tool for publishing long-form content and working with it gradually over time. Writing with any effort is deeply tied to complex thinking, and is a skill that needs to be maintained over time, and any thoughts worth writing are worth revisiting and improving over time. This site and its infrastructure serve as my platform for doing so.

1. Design

Traditional options for “blogging” leave much to be desired, especially their brittleness caused by external dependency. This site aims to be clean and focused, with features for efficiently annotating and referencing text. Static is emphasized over dynamic and minimal is emphasized over complete.

While the site emphasizes its static nature to minimize bloat and potential for script failures, there is a recognition that some trade-offs must be made for usability. The system is designed to capture the essence of static web design but with allowances for dynamic features that improve the user experience. This results in a site where both the snapshot-in-time content and the end-user experience maintain a high level of quality. Javascript is avoided where possible, and components such as footnote sections and bibliographies are rendered by the templating engine rather than dynamically using the clients resources. This ensures a consistent user experience across time and minimizes the requirements for viewing the site. Sections of articles should be displayed uniformly, even in text-based web browsers (such as Lynx).

2. Features

In an effort to enhance readability and provide additional context within text, a series of extensions to the traditional markdown format have been implemented. These extensions not only make it easier to convey complex information but also add to the overall organization of the content.

Drop Caps

D rop caps create a focal point on the page, drawing the reader’s attention to the beginning of a section or chapter. This creates a sense of visual hierarchy, emphasizing what is most important. In a page full of text, a drop cap adds a dash of style and breaks up the monotony. It adds an artistic element to what is otherwise a utilitarian arrangement of characters. Additionally, drop caps have been used historically in illuminated manuscripts and classic books. On a screen, their use helps lend a more traditional feel to a web page.

Drop caps are used in the markdown as such:

{{\< dropcap "D" >}}

Alternative Block Quotes

It is useful to have more than one type of blockquote. Regular blockquotes in markdown are very quick and easy but don’t differentiate themselves much from the rest of the text or offer a convenient way to annotate the quote author or origin. However, using a shortcode, we can add new type of blockquote:

Quote of some sort.
— Author Name: Source

This is generated in markdown as follows:

{{&lt; fancyquote  author="Author Name" source="Source" >}}
Quote of some sort. {{&lt; /fancyquote>}}

This allows us to use regular markdown blockquotes for other purposes and give actual quotes a more elegant representation.

Footnotes

Footnotes allow for a more organized reading experience by offering additional information or context without cluttering the main text. They work seamlessly in the site, appearing as clickable numbers next to the text that needs elaboration. Clicking on the number takes the reader to the Footnotes section where a more detailed explanation exists.

References

A bibliography section is automatically generated at the end of each article, providing citations for all external references used. It allows for proper credit and also serves as a resource for readers who wish to delve deeper into the topic. An archival script will attempt to store references locally as is possible.

An example of a reference to a website: {{&lt; reference content="/archives/58b18932a5357f3f93e9a4abff0eb30567beb7536ed6ef2f04bf3f7eaf3d47c3_wikipedia.org.html" >}}`.

Automatic Citations

Writing citations is annoying, mundane, and repetitive task. However, clear textual citations not only lend credit to the authors of referenced works, but also offer more information to the reader than a simple hyperlink. Fortunately, as of 2023 we live in the burgeoning age of language models, which can do a fair job at interpreting natural language and doing things like generating citations. A simple script leveraging such tools has been created alongside this site to automatically attempt to generate citations from references. The output is not perfect and has a real chance of badly mis-citing some works, but for the most part works well enough (not to mention that posts on this site are not intended to meet full academic standards for things like citations).

Only PDFs are supported as of September 2023, but I eventually would like to support citation generation from HTML as well.

Inserting Figures

Features for custom figures are also present. Alt text and figure indexes are automatically generated without client side scripts.

{{&lt; figure src="https://example.com/image.jpg" alt="Cumulus cloud rolling over the prairie" >}}

Archival

In the digital age, content is ephemeral. Websites shut down, articles are edited, and valuable information can be lost. This makes archiving a crucial component of any site concerned with long-term survival. By storing external links as archives in the site’s static data directories, I aim to combat link rot and preserve information relevant to this site’s contents. The benefits of this are not only for the site itself but also any reader who may access this content in the future. Plans are underway to also archive my own site to third-party sources to ensure its longevity.

Currently, the site supports archiving of images, PDFs, and web pages. Web pages are archived using Single File CLI, while wget is used for PDFs and images. Items using the bibliography shortcode extension are the only ones which are archived. For example, normal markdown hyperlinks will not be archived (e.g., [link description](https://example.com)). This allows the use of normal links in situations where archiving is not possible or necessary.
